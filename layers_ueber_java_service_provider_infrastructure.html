Layers über Java Service Provider Infrastructure

vorhergehende Artikel in: Geo Java Komponenten Git(lab|hub)
25.01.2019

Nachdem Berichte über Neuigkeiten in EBMap4D schon längere Zeit her sind und die letzten Artikel sich fast ausschließlich mit Tile-Servern beschäftigt haben, hier nun ein neues Update zu EBMap4D.

Bisher musste die Anwendung EBMap4D angepasst werden, wenn ein neuer Layer hinzugefügt werden sollte. Diese starre und feste Kopplung sollte abgeschafft werden (auch weil ich wieder einige Ideen für neue Layer habe). Ich habe das bisherige System aber nicht ganz fallenlassen, sondern eine weitere Möglichkeit zu den bestehenden hinzugefügt, um weitere Layer nutzen zu können.

Ich habe mich wieder einmal der guten alten Java Service Provider Infrastructure erinnert und damit gespielt. Ergebnis war eine neue Version der Anwendung EBMap4D, die es nun erlaubt, Jar-Dateien in den Klassenpfad zu legen, die der SPI entsprechen.

Damit kann man nun unabhängig von der Anwendung neue Layer entwickeln und diese dann einfach durch Kopieren des fertiggestellten Artefakts zur Anwendung hinzufügen.

Es gibt wie so oft ein Beispielprojekt auf Github, das die Möglichkeiten demonstriert.

Das war auch ein guter Zeitpunkt, die diversen Artefakte in meinem Maven-Repository zu aktualisieren...
