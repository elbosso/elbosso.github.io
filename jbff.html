JavaBeans Frontend Framework

27.04.2013

Das Framework JBFF (JavaBeans Frontend Framework) ist eine Sammlung von Komponenten und Algorithmen, die zur Laufzeit dynamisch Formulare erzeugen. Als Grundlage wird lediglich eine oder mehrere JavaBeans mit bound Properties benötigt. Es ist kein Framework-spezifischer Code nötig. Bereits bestehende JavaBeans können sofort mit JBFF benutzt werden.

JBFF kann sowohl mit nur les- wie auch mit nur schreibbaren Properties umgehen. Auch Properties, die sowohl les- wie auch schreibbar sind, stellen kein Problem dar. Für Indexed Properties gilt das bereits gesagte - hier wird in der generierten GUI eine Tabelle zum Zugriff auf die einzelnen Elemente erzeugt.

Die generierten GUI-Elemente werden abhängig vom Typ der jeweiligen Property erzeugt: Für eine Zahl-Property wird ein Eingabefeld erzeugt, das nur die Eingabe von Zahlen zuläßt, für eine boolean-Property wird eine Checkbox erstellt. Typen von Properties können nicht nur Fundamentaldatentypen sein. Auch für Klassentypen werden entsprechend korrekte Eingabeelemente erzeugt. Kann das Framework für einen Klassentypen kein Eingabeelement erzeugen - etwa weil es den Typ nicht kennt und keine Assoziation angegeben wurde, wird ein Formular für die Properties dieses Property-Typs erstellt. Die Tiefe der so entstehenden Hierarchie ist nur durch den Arbeitsspeicher begrenzt

Die erzeugten Eingabeelemente können über die BeanInfo-Klassen weiter konfiguriert werden: Beispielsweise ist es im oben genannten Beispiel möglich, einen Minimal- und einen Maximalwert für die Property anzugeben. Werden solche Validierungsregeln in der BeanInfo-Klasse angegeben, werden sie sofort bei der Eingabe angewendet und weisen den Anwender gegebenenfalls auf Fehler hin.

Das Framework wurde auf leichte Erweiterbarkeit hin optimiert: es ist problemlos möglich, Eingabeelemente für neue Property-Typen zu erstellen oder bestehende zu ersetzen.
