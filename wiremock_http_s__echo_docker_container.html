Wiremock und HTTP(s)-Echo als Docker-Container

vorhergehende Artikel in: Software-Test Linux Docker Raspi
23.09.2020

Wieder zwei Neuzugänge in meinem Docker-Zoo: dieses Mal handelt es sich um zwei Services zurt Unterstützung von Software-Tests.

Ich habe die beiden Lösungen HTTP(s)-Echo und Wiremock auf meinem Raspberry Pi 3B+ installiert - für HTTP(s)-Echo gibt es ein entsprechendes Repository, für Wiremock habe ich Dockerfile und docker-compose.yml hier unten angefügt: 

FROM ubuntu:bionic


ENV VERSION 2.27.0


#ENV HTTP_PORT 80


RUN apt-get update && apt-get -y upgrade && apt-get -y install openjdk-11-jre \
&& apt-get update && apt-get -y upgrade && apt-get -y install curl unzip \
&& mkdir /opt/wiremock


WORKDIR /opt/wiremock


RUN curl -sSL -o wiremock.jar https://repo1.maven.org/maven2/com/github/tomakehurst/wiremock-standalone/${VERSION}/wiremock-standalone-${VERSION}.jar


EXPOSE 80


ENTRYPOINT ["java","-jar","wiremock.jar","--verbose", "--port", "80"]


version: "3.1"
services:
  pi-wiremock:
    build:
      context: .
#    image: ''
    container_name: pi-wiremock
    hostname: pi-wiremock
#    ports:
#      - 8070:80
#    environment:
#        - HTTP_PORT=8888
#        - HTTPS_PORT=9999
    restart: unless-stopped
    volumes:
      - ./files:/opt/wiremock/__files
      - ./mappings:/opt/wiremock/mappings
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.wiremock.rule=Host(`wiremock.pi-docker.lab`)"
      - "traefik.http.services.wiremock.loadbalancer.server.port=80"
      - "traefik.docker.network=pi_traefik_proxy"
    networks:
      - traefik_proxy


networks:
  traefik_proxy:
    external:
      name: pi_traefik_proxy
