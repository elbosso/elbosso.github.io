Lamport Clocks für Java

vorhergehende Artikel in: Java Linux Komponenten
24.02.2024

Ich habe mich in den Weihnachtsferien hingesetzt und endlich einmal ernsthaft mit sogenannten Lamport Clocks beschäftigt, mit denen man Prozesse in verteilten Systemen synchronisieren kann.

Dazu überlegte ich mir, diese von mir angestrebte Implementierung mit der Logging-Lösung logback zu integrieren, so dass der entsprechende Zeitstempel im Mapoped Diagnostic Context (MDC) automatisch gesetzt wird und in Logmessages - entsprechende Konfiguration vorausgesetzt - automatisch mit ausgegeben wird.

Dazu habe ich die Implementierung in zwei verschiedenen Modi umgesetzt: Ich schuf eine Singleton Java-Bean ebenso wie eine Implementierung einer java.time.Clock.

Zum Testen der Implementierung erstellte ich zwei Anwendungen - eine davon als Server einer Socketverbindung und eine als Client. Inhaltlich tun beide Anwendungen dasselbe: Sie senden Events an den Kommunikationspartner und empfangen Events vom jeweils anderen.
