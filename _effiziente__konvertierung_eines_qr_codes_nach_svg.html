(Effiziente) Konvertierung eines QR-Codes nach SVG

Wie konvertiert man einen QR-Code in ein Vektorformat? Meine Antwort hier:

Seit ein paar Monaten schon wollte ich aus Java-Programmen heraus QR-Codes generieren können. Das führte unter anderem zu der Möglichkeit, QR-Codes auf einigermaßen unauffällige Weise in andere Bilder integrieren zu können.

Jedoch fand ich keine Möglichkeit, direkt SVGs daraus zu erzeugen. Da die Bibliothek zur Erzeugung der Codes zunächst erst einmal völlig wertfrei Instanzen vom Typ com.google.zxing.common.BitMatrix erstellt, muss man die auch für Bitmap-Formate erst umwandeln.

Man könnte dieses Bitmap-Bild dann nehmen und in einen GraphicsContext rendern, der von Batik zur Verfügung gestellt wird - damit erhält man dann eine SVG-Graphik: 

try
{
	org.w3c.dom.DOMImplementation domImpl = org.apache.batik.dom.svg.SVGDOMImplementation.getDOMImplementation();
	org.w3c.dom.Document document = domImpl.createDocument(org.apache.batik.dom.svg.SVGDOMImplementation.SVG_NAMESPACE_URI, "svg", null);


	int pointWidth = dimension.width;
	int pointHeight = dimension.height;


	org.apache.batik.svggen.SVGGraphics2D svgGenerator = new org.apache.batik.svggen.SVGGraphics2D(document);


	svgGenerator.setSVGCanvasSize(new java.awt.Dimension(pointWidth, pointHeight));
	g2d = svgGenerator;
	g2d.setColor(Color.BLACK);
	g2d.fillRect(0, 0, dimension.width, dimension.height);


	g2d.setColor(Color.WHITE);


	//in GraphicsContext zeichnen
	
	java.io.FileOutputStream bout = new java.io.FileOutputStream("/tmp/qr.svg");
	java.io.Writer out = new java.io.OutputStreamWriter(bout, "UTF-8");
	svgGenerator.stream(out, false);
	out.close();
	bout.close();
}
catch (java.lang.Throwable t)
{
	t.printStackTrace();
}
Das bedeutet aber, dass die Bilddaten als Rasterimage in das SVG eingebunden werden - und sie damit nicht mehr verlustfrei skaliert werden können. Daher kann man den Zwischenschritt der Erzeugung der Bitmap überspringen und direkt in den GraphicsContext zeichnen.

Damit bekommt man ein SVG, das verlustfrei skalierbar ist. Allerdings ist es sehr groß: für jeden Pixel wird ein eigenes Element in das SVG eingebaut. Wenn man herausfinden könnt, welche der Elemente zusammenhängen und aus diesen ein Element machen könnte - dann würde sich der Aufwand zum Zeichnen und ganz nebenbei auch die Größe der Datei erheblich reduzieren lassen.

Java hat viele Bibliotheken an Bord - daher dachte ich mir: Das muss doch mit Bordmitteln gehen? Siehe da - es geht: Man kann herausfinden, ob zwei Areas sich überlappen und falls dem so ist, beide vereinigen: 

//com.google.zxing.common.BitMatrix bitMatrix=...;
java.util.LinkedList<java.util.List<java.awt.Shape>> shapes = new java.util.LinkedList();
for (int i = 0; i < dimension.width; i++)
{
	int x = -1;
	int y = -1;
	int l = 0;
	java.util.List<java.awt.Shape> list = new java.util.LinkedList();
	shapes.addLast(list);
	for (int j = 0; j < dimension.height; j++)
	{
		if (bitMatrix.get(i, j) == false)
		{
			if (x < 0)
			{
				x = i;
				y = j;
			}
			++l;
		}
		else
		{
			if (x > -1)
			{
				java.awt.Shape p = createShape(x, y, l);
				list.add(p);
				x = -1;
				y = -1;
				l = 0;
			}
		}
	}
	if (x > -1)
	{
		java.awt.Shape p = createShape(x, y, l);
		list.add(p);
	}
}
int end = shapes.size() / 2;
for (int i = 0; i < end; ++i)
{
	java.util.LinkedList<java.util.List<java.awt.Shape>> old = shapes;
	shapes = new java.util.LinkedList();


	while (old.size() > 2)
	{
		java.util.List<java.awt.Shape> l = old.removeFirst();
		java.util.List<java.awt.Shape> r = old.removeFirst();
		java.util.List<java.awt.Shape> n = new java.util.LinkedList();
		java.util.List<java.awt.Shape> toberem = new java.util.LinkedList();
		for (Shape shapel : l)
		{
			for (Shape shaper : r)
			{
				if (shapel.intersects(shaper.getBounds2D()))
				{
					toberem.add(shaper);
					java.awt.geom.Area areal = new java.awt.geom.Area(shapel);
					java.awt.geom.Area arear = new java.awt.geom.Area(shaper);
					areal.add(arear);
					shapel = areal;
				}
			}
			n.add(shapel);
			r.removeAll(toberem);
			toberem.clear();
		}
		n.addAll(r);
		shapes.add(n);
	}
	shapes.addAll(old);
}
Wenn man die Erzeugung des Polygons ein wenig geschickt anstellt, findet man heraus, dass damit tatsächlich eine sehr viel kompaktere Möglichkeit der Darstellung verlustfrei skalierbarer QR-Codes herauskommt. 
private java.awt.Shape createShape(double x, double y, double l)
{
	double e = 0.001;
	java.awt.geom.Path2D.Double p = new java.awt.geom.Path2D.Double();
	p.moveTo(x - e, y - e);
	p.lineTo(x + 1 + e, y - e);
	p.lineTo(x + 1 + e, y + l + e);
	p.lineTo(x - e, y + l + e);
	p.closePath();
	return p;
}
Hier zwei Beispiele: Beide repräsentieren denselben QR-Code, eine davon ist unkomprimiert und eine zeigt die Möglichkeiten, wenn zusammenhängende Polygone vereinigt werden.
