Maven-Projekt für Module für dWb+

vorhergehende Artikel in: Java dWb+ Komponenten Git(lab|hub)
23.01.2016

Nachdem ich ein Maven Repository für meine Java-Komponenten eingerichtet habe, hier nun eine beispielhafte Konfiguration eines Maven-Projektes zur Erstellung von Modulen für dWb+.

Hier ein kleines Beispiel-pom für ein Projekt, das Module für dWb+ beinhaltet. 

&lt;project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/maven-v4_0_0.xsd">
  &lt;modelVersion>4.0.0&lt;/modelVersion>
  &lt;groupId>de.elbosso&lt;/groupId>
  &lt;artifactId>custom_maven_modules&lt;/artifactId>
  &lt;packaging>jar&lt;/packaging>
  &lt;version>1.0-SNAPSHOT&lt;/version>
  &lt;name>Custom Maven Modules for dWb+&lt;/name>
  &lt;url>http://maven.apache.org&lt;/url>
   &lt;build>
      &lt;pluginManagement>
         &lt;plugins>
            &lt;plugin>
               &lt;groupId>org.apache.maven.plugins&lt;/groupId>
               &lt;artifactId>maven-compiler-plugin&lt;/artifactId>
               &lt;version>2.5.1&lt;/version>
            &lt;/plugin>
         &lt;/plugins>
      &lt;/pluginManagement>
   &lt;/build>
   &lt;properties>
		&lt;project.custom.encoding>UTF-8&lt;/project.custom.encoding>
		&lt;project.custom.java.version>1.6&lt;/project.custom.java.version>
		&lt;maven.compiler.source>${project.custom.java.version}&lt;/maven.compiler.source>
		&lt;maven.compiler.target>${project.custom.java.version}&lt;/maven.compiler.target>
		&lt;project.build.sourceEncoding>${project.custom.encoding}&lt;/project.build.sourceEncoding>
		&lt;project.build.outputEncoding>${project.custom.encoding}&lt;/project.build.outputEncoding>
		&lt;project.reporting.outputEncoding>${project.custom.encoding}&lt;/project.reporting.outputEncoding>
   &lt;/properties>
	&lt;repositories>
		&lt;repository>
			&lt;releases>
				&lt;enabled>true&lt;/enabled>
				&lt;updatePolicy>always&lt;/updatePolicy>
				&lt;checksumPolicy>fail&lt;/checksumPolicy>
			&lt;/releases>
			&lt;id>YourId&lt;/id>
			&lt;name>YourName&lt;/name>
			&lt;url>http://elbosso.github.io/maven2&lt;/url>
			&lt;layout>default&lt;/layout>
		&lt;/repository>
        &lt;repository>
          &lt;id>central&lt;/id>
          &lt;url>http://repo1.maven.org/maven2&lt;/url>
          &lt;releases>
            &lt;enabled>true&lt;/enabled>
          &lt;/releases>
        &lt;/repository>
	&lt;/repositories>
  &lt;dependencies>
    &lt;dependency>
      &lt;groupId>de.elbosso&lt;/groupId>
      &lt;artifactId>dataflowframework_api&lt;/artifactId>
      &lt;version>0.1.0-SNAPSHOT&lt;/version>
    &lt;/dependency>
  &lt;/dependencies>
&lt;/project>

Das entstehende JAR einfach in das Verzeichnis kopieren, das alle anderen Modul-JARs enthält - danach kann man mit den neuen Modulen sofort loslegen...
