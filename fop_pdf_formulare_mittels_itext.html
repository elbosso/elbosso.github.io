FOP und PDF-Formulare mittels iText

vorhergehende Artikel in: Java Komponenten Markup
08.09.2016

Der Editor für Testprozeduren und Checklisten verfügte bereits seit längerer Zeit über eine Vielzahl von Features - allerdings mit dem Fokus auf Protokolle, die in Papierform erzeugt und ausgefüllt wurden. Das hat sich nun geändert...

Im vorangehenden Artikel schilderte ich Ideen, wie PDF-Dokumente, die mittels Apache FOP erzeugt werden, mit PDF-Formularelementen angereichert werden könnten. Ich habe eine dieser Ideen umgesetzt und präsentiere hier die Resultate - zunächst eine Testprozedur, wie sie der Editor für Testprozeduren und Checklisten bisher erzeugte und auch weiterhin erzeugen wird:

Original
Die Anwendung wurde um die Möglichkeit erweitert, ein neues Ausgabeformat zu exportieren. Dabei handelt es sich um ein PDF-Dokument, das allerdings an den wichtigen Stellen wiedererkennbare Platzhalter aufweist und daher dringend nachbearbeitet werden muss:
Generiertes PDF-Dokument mit Platzhaltern
Und hier schließlich das Ergebnis, nachdem das erzeugte Zwischendokument analysiert wurde und die Platzhalter darin durch entsprechende Formularelemente ersetzt wurden.
Nachbearbeitetes PDF-Dokument mit Formularelementen
Der Code dafür ist natürlich noch sehr speziell auf das vorliegende Problem zugeschnitten. Dafür funktioniert er aber ausreichend gut und man kann darauf aufbauen, um andere ähnliche Projekte umzusetzen...
