Code zur Versionierung von Datenbank-Schemata

vorhergehende Artikel in: Datenbanken Git(lab|hub)
13.03.2018

Manchmal muss man für eine dezentrale Anwendung ein Datenbank-Schema entwerfen, von dem man weiß. dass es sich während der Entwicklung der zugehörigen Applikation ändern wird. Es wäre schön, wenn die Anwendung dazu einen Mechanismus eingebaut hätte, der das Datenbank-Schema automatisch auf die neueste Version anpasst.

Ich habe dazu einmal eine kleine Klasse - eigentlich nur eine Funktion geschrieben, die als Grundlage oder Inspiration einer solchen production-ready Funktionalität dienen kann und diesen Code auf GitHub als Gist publiziert.

Und ich weiß natürlich, dass es Dinge wie Liquibase gibt - aber wenn man lernen will, muss man Dinge tun, damit Fehler geschehen, aus denen man dann lernen kann...

Dieser Code wurde zum Beispiel im Demonstrator für einen sichereren Webbrowser eingesetzt.

Es existiert auch ein entsprechendes Maven-Fragment, das wie folgt eingebunden werden kann: 

<!--...-->
    <repositories>
        <repository>
            <releases>
                <enabled>true</enabled>
                <updatePolicy>always</updatePolicy>
                <checksumPolicy>fail</checksumPolicy>
            </releases>
            <id>YourId</id>
            <name>YourName</name>
            <url>http://elbosso.github.io/maven2</url>
            <layout>default</layout>
        </repository>
<!--...-->
    </repositories>
<!--...-->
    <dependencies>
        <dependency>
            <groupId>de.elbosso</groupId>
            <artifactId>domains</artifactId>
            <version>0.1.0-SNAPSHOT</version>
        </dependency>
<!--...-->
    </dependencies>
<!--...-->
