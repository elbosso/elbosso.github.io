DigiSparkBlinkStick II

vorhergehende Artikel in: Hardware Git(lab|hub)
25.07.2020

Ich habe vor kurzem darüber berichtet, wie ich meinen eigenen BlinkStick realisiert habe

Der BlinkStick aus eigener Produktion funktioniert inzwischen sehr gut - ich habe wie bereits beschrieben sowohl Muster mit einer einfachen RGB-Led, wie auch Muster mit einer "smarten" Led im Einsatz.

Bisher hatte ich die MQTT-Unterstützung nur begonnen - man konnte noch lange nicht von einem operativen Einsatz sprechen. Das hat sich nun geändert: Ich habe das Skript zum Lauschen auf MQTT-Nachrichten angepasst und auch einen SystemD-Service spendiert, um dieses Skript als Daemon laufen zu lassen.

Die Installation gestaltet sich denkbar einfach (alle folgenden Befehle werden mit Root-Rechten ausgeführt: 

apt-get update
apt-get upgrade
apt-get install joe mosquitto-clients
cd /opt
git clone https://github.com/elbosso/DigiSparkBlinkStick.git
joe /etc/systemd/system/blinkstickmqtt.service

Der Text der Datei muss wie folgt lauten: 

[Unit]
After=network.target


[Service]
ExecStart=/opt/DigiSparkBlinkStick/scripts/digispark_mqtt.sh


[Install]
WantedBy=default.target

Anschließend wird der Dienst aktiviert: 

systemctl daemon-reload
systemctl enable blinkstickmqtt.service
service blinkstickmqtt start

Anschließend können von einem beliebigen Rechner aus Botschaften an den Broker gesendet werden und alle Blinksticks an Maschinen, auf denen dieser Dienst läuft werden entsprchend reagieren. So würde zum Beispiel das folgende Kommando bewirken, dass die entsprechenden BlinkSticks nach 2 Stunden violett leuchten: 

echo mosquitto_pub -h mqtt.pi-docker.lab -t home/blinkstick/raw -m "purple" | at now + 120 minute
